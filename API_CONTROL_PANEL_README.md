# Панель управления API серверами Sportsense AI

## 🎯 Назначение

Tkinter GUI приложение для управления API серверами и тестирования функций без запуска основного Flutter приложения.

## 🚀 Запуск

```bash
python scripts/api_control_panel.py
```

> ⚠️ Если psutil не установлен, некоторые функции будут использовать альтернативные методы (netstat/taskkill).

## 📋 Функции

### 1. Управление серверами

#### 🤖 Qwen LLM API (порт 5000)
- **Запуск/Остановка** - кнопка переключения
- **Проверить** - проверка здоровья сервера
- Статус отображается в реальном времени:
  - ● Остановлен (красный)
  - ● Работает (зеленый)
  - ● Работает (модель не загружена) (желтый)

#### ⚽ UEFA Parser API (порт 5001)
- **Запуск/Остановка** - кнопка переключения
- **Проверить** - проверка здоровья сервера
- Показывает наличие кэша

#### 🛑 Остановить все Python процессы
- Останавливает ВСЕ Python процессы (кроме текущего)
- Требует подтверждения

#### 🔄 Обновить статус
- Обновляет статус обоих серверов

### 2. Тестирование функций

#### 📝 Генерация названия чата
- Введите любой вопрос
- Нажмите "Сгенерировать"
- Результат:
  - **Локальная генерация** - работает всегда (на основе ключевых слов)
  - **LLM генерация** - только если Qwen API запущен

**Примеры:**
- "Расскажи про футбол" → "💬 Вопрос про футбол"
- "Как работает нейросеть?" → "💬 Вопрос про нейросети"
- "Рецепт борща" → "💬 Вопрос про еда"

#### 💬 Тест чата с LLM
- Введите сообщение
- Нажмите "Отправить"
- Получите ответ от модели
- Показывает количество использованных токенов

> ⚠️ Требует запущенного Qwen API сервера!

#### 🏆 Тест UEFA парсера
- **Получить рейтинги** - данные из кэша
- **Свежие данные** - парсинг с сайта UEFA (занимает время)

> ⚠️ Требует запущенного UEFA Parser API сервера!

### 3. Лог
- Реальный-time лог всех операций
- Цветовая индикация:
  - ✅ Успех (зеленый)
  - ❌ Ошибка (красный)
  - ⚠️ Предупреждение (желтый)
  - ℹ️ Информация (синий)
- **Очистить лог** - очистка содержимого

## 🔧 API Endpoints

### Qwen API (порт 5000)

#### GET /health
Проверка доступности сервера.

**Ответ:**
```json
{
  "status": "ok",
  "model": "Qwen2.5-1.5B-Instruct",
  "loaded": true
}
```

#### POST /chat
Чат с моделью (с системным промптом).

**Запрос:**
```json
{
  "message": "Привет!",
  "max_tokens": 256,
  "temperature": 0.7
}
```

**Ответ:**
```json
{
  "response": "Привет! Как дела?",
  "model": "Qwen2.5-1.5B-Instruct",
  "tokens_used": 15
}
```

#### POST /generate
Генерация текста (без системного промпта).

**Запрос:**
```json
{
  "prompt": "Напиши стих про кота",
  "max_tokens": 100
}
```

**Ответ:**
```json
{
  "text": "Котик милый, котик...",
  "tokens": 45
}
```

### UEFA Parser API (порт 5001)

#### GET /health
Проверка доступности сервера.

**Ответ:**
```json
{
  "status": "ok",
  "service": "UEFA Parser API",
  "cached": true,
  "cache_timestamp": "2024-01-01T12:00:00"
}
```

#### GET /rankings
Получение данных рейтинга (из кэша если есть).

**Ответ:**
```json
{
  "status": "ok",
  "source": "cache",
  "data": [
    {
      "rank": "1",
      "association": "England",
      "points": "100.000"
    }
  ],
  "count": 50,
  "timestamp": "2024-01-01T12:00:00"
}
```

#### GET /rankings/fresh
Получение свежих данных (парсинг с сайта).

**Ответ:** Аналогично `/rankings`, но `source: "fresh"`

## 🐛 Отладка

### Сервер не запускается
1. Проверьте что порт не занят: `netstat -ano | findstr :5000`
2. Убейте процесс: `taskkill /F /PID <PID>`
3. Проверьте зависимости: `pip install flask flask-cors llama-cpp-python`

### Модель не загружается
1. Проверьте наличие файла модели: `models/qwen2.5-1.5b-instruct-gguf/`
2. Если нет - запустите: `python scripts/download_qwen.py`

### Ошибка "Connection refused"
- Сервер не запущен
- Неправильный порт
- Фаервол блокирует соединение

## 💡 Советы

1. **Тестируйте генерацию названий** без запуска LLM - локальный алгоритм работает всегда
2. **Используйте "Проверить"** для диагностики проблем
3. **Следите за логом** для отладки
4. **Останавливайте серверы** перед закрытием приложения
